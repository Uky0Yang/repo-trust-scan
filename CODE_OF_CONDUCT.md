# Code of Conduct

## Our standard

We want this project to be a useful, welcoming place for security-minded developers. Be respectful, discuss observable behavior rather than presumed intent, and give contributors room to correct mistakes.

Helpful participation includes:

- Sharing minimal, non-sensitive reproductions
- Explaining false positives with technical context
- Critiquing code and claims rather than people
- Respecting disclosure boundaries and contributor privacy

Harassment, discriminatory language, personal attacks, publishing private information, and knowingly submitting live credentials or harmful payloads are not acceptable.

## Security-sensitive material

Do not post real tokens, private repository content, or weaponized samples in public issues. Follow [SECURITY.md](SECURITY.md) for private reporting.

## Enforcement

Project maintainers may edit or remove content, close interactions, or restrict participation when behavior conflicts with this standard. Enforcement decisions should be proportionate, documented where appropriate, and focused on keeping the project safe and constructive.
